# 功能优先级（新手视角）

面向当前仓库的知识差距文档，按「先易后难、收益靠前」排序，便于 Java 新手落地。

## 1. 先做（基础设施 + 核心支付）
- 数据与监控：优先加 Actuator/Micrometer、traceId 透传，便于后续排查（参考 `knowledge-gap-reports/数据与监控/需求文档.md`、`设计文档.md`）。
- 付费体系 M1/M2：实现钱包、模拟充值、余额查询、章节购买/授权、幂等与事务（参考 `knowledge-gap-reports/paid-system-docs/2025-12-27_17-21-29.md`）。可练习 BigDecimal、行锁/版本号、@Transactional。

## 2. 支撑型能力（中等难度）
- 通知中心：先做站内信 + WebSocket 未读数，复用现有邮件兜底，支撑支付结果/审核回执等（参考 `knowledge-gap-reports/通知中心/需求文档.md`、`设计文档.md`）。
- 内容安全与审核：先搭骨架 + 本地敏感词拦截 + 队列占位，第三方审核稍后接入（参考 `knowledge-gap-reports/内容安全与审核/需求文档.md`、`设计文档.md`）。

## 3. 功能扩展（需要新组件/依赖）
- #### 搜索与推荐：落地 ES/Redis 前的基础搜索 + 热榜缓存，再逐步加自动补全与埋点驱动推荐（参考 `knowledge-gap-reports/搜索与推荐/需求文档.md`、`设计文档.md`）。
- 活动与运营：等付费/通知稳定后，再做限时免费 + 基础发券；抽奖/弹窗放在后面（参考 `knowledge-gap-reports/活动与运营/需求文档.md`、`设计文档.md`）。

## 4. 高级阶段（外部依赖多，最后做）
- 付费体系 M3/M4：对账、限流/风控、监控告警、真实支付、分账、提现、优惠券/限免等在基础支付稳定后推进（参考 `knowledge-gap-reports/paid-system-docs/paid-system-m3-m4-requirements.md`、`paid-system-m3-m4-design.md`）。
